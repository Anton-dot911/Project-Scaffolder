# ТЗ №4 — Project Scaffolder (внутрішній інструмент)

**Кодова назва:** `antlab-create`
**Тип:** CLI для особистого користування
**Пріоритет:** Етап 0 (перший інструмент)
**Оцінка тривалості:** 2–3 дні (мінімальна версія — 1 день)

---

## 1. Призначення

Кожен новий проєкт починається з тих самих 2–4 годин рутини: структура папок, конфіги, лінтери, env, CI, git. Скаффолдер зводить це до однієї команди і гарантує, що фундамент кожного проєкту відповідає спільним стандартам (див. Roadmap §3).

## 2. Функціональні вимоги

### Мінімальна версія (день 1)
- **F1. Запуск:** `npx antlab-create my-project` → інтерактивний вибір шаблону.
- **F2. Три шаблони:**
  - `ts-fullstack`: React+Vite+TS+Tailwind фронтенд, Node/Fastify бекенд, спільні типи (pnpm workspace);
  - `py-service`: FastAPI + Pydantic v2 + pytest + ruff + mypy + Dockerfile;
  - `ts-cli`: Node CLI з commander, tsup-збіркою.
- **F3. Кожен шаблон містить:** `.env.example`, README-каркас (EN, зі стандартними секціями), ESLint/Prettier або ruff-конфіг, GitHub Actions (lint+test), `.gitignore`, git init + перший коміт.
- **F4. AI-шар у шаблонах** (для fullstack/py-service): готовий модуль `llm/` — обгортка Anthropic SDK з retry/backoff, Zod/Pydantic-валідацією structured outputs, каталогом `/prompts`, інтеграцією з Cost/Latency Tracker (ТЗ №6).

### v1.1 (за потребою)
- F5. Прапорці для неінтерактивного запуску (`--template py-service --no-git`).
- F6. Опційні модулі: `--with supabase` (клієнт+міграції), `--with evals` (каркас eval-скрипта під Golden Dataset Builder).
- F7. `antlab-create update-shared` — синхронізація конфігів лінтерів у існуючих проєктах.

## 3. Технічні рішення

- Node/TypeScript, шаблони — каталоги з файлами + плейсхолдери (`{{project_name}}`); проста підстановка, без складних template-движків.
- Розповсюдження: приватний GitHub-репозиторій, запуск через `npx github:antlab/antlab-create` (публікація в npm — опційно пізніше).
- Тест: e2e — генерація кожного шаблону в temp-каталог → встановлення залежностей → `lint` і `test` проходять.

## 4. Definition of Done

- [ ] Один командний запуск дає проєкт, у якому `dev`, `lint`, `test` працюють одразу
- [ ] Модуль `llm/` у шаблонах пройшов смоук-тест реальним викликом API
- [ ] DocFlow (ТЗ №1) реально розпочато зі скаффолдера — це і є приймальний тест
